# ADR 0009: Parallel Agents via Isolated Execution Contexts

## Status
Accepted

## Date
2026-03-22

## Context

Мультиагентность полезна только тогда, когда подзадачи реально можно изолировать. Без изоляции несколько агентов начинают конфликтовать по файлам, дублировать решения и усложнять ревью.

Для CodeCore нужен способ параллелить работу без превращения системы в хаотичный диспетчер агентов.

## Decision

Параллельные агенты допускаются только через изолированные execution contexts.

Предпочтительная стратегия:

- отдельные worktrees/ветки;
- отдельный context snapshot;
- отдельные tool budgets и policy scopes;
- явная роль агента.

Базовые роли параллельной работы:

- researcher;
- planner;
- coder;
- reviewer.

Правила:

- один агент не должен неявно менять файлы другого агента;
- результаты параллельной работы собираются через diff and merge discipline;
- reviewer не должен быть тем же execution context, что и автор change set, если требуется независимая проверка.

## Consequences

Плюсы:

- реальная мультиагентность становится управляемой;
- проще анализировать вклад и ошибки каждого агента;
- легче использовать worktree-based workflows и isolated experiments;
- меньше конфликтов и несогласованных правок.

Минусы:

- выше стоимость orchestration;
- нужен merge flow и статусная панель по execution contexts;
- параллельность не бесплатна и не нужна для простых задач.

Принятый компромисс:

- single-agent остаётся базовым режимом;
- multi-agent включается только для задач, где есть выигрыш от изоляции ролей.
